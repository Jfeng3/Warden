# Product Strategy (docs/product-strategy/)

Canonical source of truth for messaging, customer segmentation, and competitive differentiation.

## Rules

- **Do not modify files in `docs/product-strategy/` without explicit user approval** — these documents drive landing page copy, skill definitions, and cron job instructions. Changes have downstream effects.
- When referencing product positioning, always read the current strategy docs first rather than relying on memory — messaging evolves.
- Skills and content workflows should align with these docs but not duplicate them. Reference the strategy; don't copy it inline.

## Key Files

- `messaging.md` — One-liner, headline, core narrative, proof points, messaging constraints
- `customer-segmentation.md` — Primary persona (solo operators) and secondary (small team leads 2-5)
- `differentiation.md` — Competitive positioning vs freelancers, agencies, Jasper/Copy.ai, hiring, doing nothing. AEO-native content as unique edge.
